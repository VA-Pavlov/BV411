### Stepan prodg
## README — Веб‑сайт на ASP.NET

### Краткое описание
Простой веб‑сайт на ASP.NET, реализующий статические и динамические страницы с использованием HTML и CSS. Сервер отвечает за рендеринг страниц и обработку минимальной логики (формы, маршрутизация).

### Установка (локальная)
1. Клонируйте репозиторий:
   ```bash
   git clone <repo-url>
   ```
2. Перейдите в папку проекта:
   ```bash
   cd <repo-folder>
   ```
3. Восстановите зависимости и соберите проект:
   ```bash
   dotnet restore
   dotnet build
   ```
4. Запустите приложение:
   ```bash
   dotnet run --project <путь-к-проекту>
   ```

### Режим разработки
- Быстрый запуск с автообновлением:
  ```bash
  dotnet watch run --project <путь-к-проекту>
  ```
- Статические файлы (HTML, CSS, изображения, JS) размещаются в `wwwroot` или в папке представлений по соглашению проекта.
- Маршруты и страницы настраиваются в `Program.cs` / `Startup.cs` (в зависимости от шаблона).

### Конфигурация
- Основные настройки можно хранить в `appsettings.json`.
- Для переключения окружения используйте переменную:
  ```bash
  ASPNETCORE_ENVIRONMENT=Development
  ```
- Не храните секреты в репозитории — используйте переменные окружения при необходимости.



### CI / CD (рекомендации)
- В pipeline выполняйте шаги:
  1. `dotnet restore`
  2. `dotnet build`
  3. `dotnet test`
  4. `dotnet publish` (если требуется артефакт для деплоя)

### Безопасность и рекомендации
- В продакшн обязательно включите HTTPS.
- Ограничьте публичный доступ к конфигурациям и секретам.
- Следите за обновлениями пакетов и уязвимостями.

### Вклад в проект
1. Форкните репозиторий и создайте ветку:
   ```
   feature/<описание>
   ```
2. Внесите изменения и добавьте тесты (если применимо).
3. Откройте Pull Request с описанием изменений и назначьте ревью.
